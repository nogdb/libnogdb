# Code Review Checklist

Follow this checklist when reviewing code.

## Security
- [ ] Input validation
- [ ] SQL injection prevention
- [ ] XSS prevention
- [ ] Authentication/authorization checks

## Performance
- [ ] No N+1 queries
- [ ] Proper indexing
- [ ] Caching where appropriate
- [ ] Response time acceptable

## Quality
- [ ] Tests included
- [ ] Documentation updated
- [ ] Error handling
- [ ] Logging added

## License Compliance
- [ ] No licensed code exposed to AI
- [ ] Interfaces used for licensed integrations
- [ ] Adapters properly implemented

## Activation
- **Mode**: Model Decision
- **Description**: Apply when reviewing pull requests or code changes
